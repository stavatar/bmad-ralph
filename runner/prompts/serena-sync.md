## Инструкции

1. Вызови `list_memories` чтобы увидеть все существующие memories.
2. Вызови `read_memory` для каждой memory, которая может быть затронута изменениями.
3. Обнови затронутые memories через `edit_memory` (предпочтительно) или `write_memory` (только для новых).
4. Убедись что каждая memory отражает актуальное состояние кодовой базы.

## Ограничения

- ЗАПРЕЩЕНО удалять memories (`delete_memory`).
- ЗАПРЕЩЕНО создавать memories без явной необходимости — предпочитай обновление существующих.
- Предпочитай `edit_memory` над `write_memory` для существующих memories.
- Не добавляй спекулятивную или непроверенную информацию.
- Каждое изменение должно быть обосновано конкретным diff или уроком.
