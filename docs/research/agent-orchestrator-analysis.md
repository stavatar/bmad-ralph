# ComposioHQ/agent-orchestrator — Анализ для bmad-ralph

## Дата: 2026-03-02
## Контекст: Сравнение подходов к управлению знаниями и оркестрации AI-агентов

## Что это такое

**agent-orchestrator** (ComposioHQ) — система оркестрации параллельных AI-агентов (fleet orchestrator). Управляет несколькими агентами одновременно: распределяет задачи, координирует, агрегирует результаты.

## Ключевые характеристики

| Аспект | agent-orchestrator | bmad-ralph |
|--------|-------------------|------------|
| Цель | Параллельная оркестрация флота агентов | Качество одиночной сессии + накопление знаний |
| Знания | НЕТ системы знаний | Полный pipeline: извлечение → дистилляция → инъекция |
| Обучение | НЕТ обучения между сессиями | Каждая сессия обогащает базу знаний |
| Память | Stateless — каждая сессия с нуля | Stateful — DistillState, LEARNINGS.md, ralph-*.md |
| Правила | Шаблонные (5-7 на язык) | 122+ конкретных паттернов из code review |
| Quality gates | НЕТ | G1-G6 (валидация, code review, human gates) |
| Violation tracking | НЕТ | Полный цикл с escalation thresholds |

## Вывод для bmad-ralph

**agent-orchestrator НЕ конкурент, а потенциально комплементарный инструмент.**

- Сильная сторона AO: управление флотом (запуск N агентов параллельно, координация)
- Сильная сторона bmad-ralph: качество каждой сессии (знания, quality gates, violation tracking)

Пересечений в системе управления знаниями — НОЛЬ. agent-orchestrator не имеет:
- Извлечения знаний из сессий
- Дистилляции/обобщения
- Инъекции знаний в промпты
- Скоупированных правил (.claude/rules/*.md с globs)
- Обратной связи через violation tracking

## Релевантность для Epic 6

Исследование подтверждает: автоматическая генерация scoped rules через LLM-дистилляцию — уникальная функциональность bmad-ralph. Ни agent-orchestrator, ни другие изученные проекты не реализуют аналогичный подход.
