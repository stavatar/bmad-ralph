# Research: Метрики качества ralph plan

## Определение "хорошего плана"

Хороший `sprint-tasks.md` — это файл, который ralph run может автономно выполнить без ручного вмешательства. Измеримые критерии:

### 1. Структурная корректность
- **Парсится сканером**: `runner.ScanTasks()` находит >= 1 задачу
- **Маркеры задач**: каждая задача начинается с `- [ ]` (TaskOpen)
- **Нет невалидных маркеров**: отсутствуют строки типа `- []`, `- [X]` (uppercase), `- [  ]`

### 2. Полнота покрытия
- **Все AC покрыты**: каждый acceptance criterion из story-файлов представлен хотя бы в одной задаче
- **Количество задач >= baseline**: не меньше чем предыдущий bridge output (для сравнения)
- **Нет пустых задач**: каждая задача содержит описание > 10 символов

### 3. Source-ссылки (FR20)
- **Битые ссылки = 0**: все `[Source: file.md#Section]` ссылки резолвятся
- **Typed headers**: роли файлов (PRD, Architecture, etc.) корректно отображены

### 4. Исполняемость
- **Задачи самодостаточны**: каждая задача содержит достаточно контекста для выполнения Claude Code агентом
- **Нет циклических зависимостей**: порядок задач логически последователен
- **Subtask-уровень**: нет задач слишком крупных (> 500 слов описания) или слишком мелких (< 3 слова)

## Как автоматически измерить качество

### Метрики уровня 1 (полностью автоматические)

| Метрика | Формула | Порог |
|---------|---------|-------|
| task_count | `strings.Count(content, TaskOpen)` | >= 10 |
| valid_markers | отсутствие невалидных маркеров | 0 invalid |
| parseable | `ScanTasks()` без ошибок | true |
| avg_task_length | среднее кол-во символов в описании задачи | >= 50 |
| broken_sources | grep `[Source:` -> проверка существования файла | 0 |
| empty_tasks | задачи с описанием < 10 символов | 0 |

### Метрики уровня 2 (полуавтоматические, LLM-as-judge)

| Метрика | Метод | Порог |
|---------|-------|-------|
| ac_coverage | LLM сравнивает AC из story с задачами | >= 90% |
| task_clarity | LLM оценивает самодостаточность каждой задачи | >= 80% задач "clear" |
| scope_match | LLM проверяет, нет ли scope creep | 0 out-of-scope задач |

### Метрики уровня 3 (runtime, после выполнения)

| Метрика | Источник | Порог |
|---------|----------|-------|
| execution_success_rate | ralph run: доля задач без retry | >= 70% |
| review_pass_rate | ralph run: доля чистых review | >= 60% |
| emergency_stop_rate | ralph run: emergency stops | 0 |

## Пороговые значения для входа в Growth

### Минимальные пороги (все должны быть выполнены)

1. **task_count >= 10** — план содержит достаточно задач
2. **broken_sources = 0** — все ссылки валидны
3. **parseable = true** — ScanTasks парсит без ошибок
4. **empty_tasks = 0** — нет пустых задач

### Рекомендуемые пороги (желательно)

1. **avg_task_length >= 50** символов
2. **ac_coverage >= 90%** (требует LLM-as-judge)
3. **Время генерации < 120 сек** — разумная стоимость

## Рекомендация

**Продолжать Growth фичи** при условии:
- Story 15.1 acceptance test пройден (plan генерирует валидный output)
- Минимальные пороги уровня 1 выполнены
- broken_sources = 0 подтверждён на реальном проекте

Growth фичи (replan, init) не зависят от метрик уровня 2-3 — эти метрики станут актуальны при масштабировании ralph plan на множество проектов.

Метрики уровня 2-3 рекомендуется реализовать как отдельный epic после завершения Growth, когда появится достаточно данных для калибровки порогов.
