### Дополнительно: Knowledge Extraction Enforcement

Реализованы 2 механизма для принудительного извлечения знаний после code-review:

**1. Verification script** (`.claude/hooks/verify-knowledge-extraction.sh`):
- Проверяет через `git diff` + `git ls-files --others` что `.claude/rules/` и `.claude/violation-tracker.md` были реально изменены
- Информационный (exit 0), выводит OK/WARN по каждой категории + итоговый RESULT

**2. Усиление Step 6 в code-review workflow** (`instructions.xml`):
- 5 структурированных подшагов вместо общего "analyze findings":
  - 6a: Категоризация findings (новый паттерн vs существующий)
  - 6b: Новые паттерны → конкретный topic file с citation
  - 6c: Обновление violation-tracker.md (MANDATORY даже без новых паттернов)
  - 6d: Обновление memory/MEMORY.md
  - 6e: Запуск `verify-knowledge-extraction.sh` — если INCOMPLETE, блокирует завершение ревью
- Структурированный output с таблицей violation counts
