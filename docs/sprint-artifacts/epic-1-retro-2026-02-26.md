## Проблемы и области роста

1. **Повторяющиеся находки рецензий.** Дублирование тестов обнаружено в историях 1.3, 1.4 и 1.12 — одна и та же ошибка трижды. Документирование правила в CLAUDE.md недостаточно для предотвращения.
2. **Трение Windows/WSL.** Концы строк (CRLF), пути Go, разная семантика os.MkdirAll — накладные расходы в каждой истории. Решения найдены, но отнимали время.
3. **Потеря контекста между историями.** Знания, извлечённые из рецензии истории 1.10 (маркеры обрезки шаблонов), не были применены в истории 1.12. Разрыв между «документировано» и «применяется».

## Ключевые находки и уроки

### По тестированию
- Именование тестов: строго `Test<Тип>_<Метод>_<Сценарий>`, где «Тип» — реальный тип Go
- Всегда тестировать нулевые значения пользовательских типов ошибок
- `errors.As` вместо приведения типов — будущая устойчивость
- Проверять содержание сообщения об ошибке, не только `err != nil`
- `t.Logf` — НЕ ломает тест, только `t.Errorf`/`t.Fatalf`
- Каждая экспортируемая функция требует выделенного теста на ошибки
- Windows: `os.UserHomeDir()` использует `USERPROFILE`, не `HOME`

### По архитектуре
- `config` — листовой пакет, зависит только от stdlib
- Sentinel-ошибки: `var ErrNoTasks = errors.New("нет задач")`
- Оборачивание ошибок: `fmt.Errorf("пакет: операция: %w", err)`
- `text/template` `missingkey=error` работает только для словарей, не для структур
- `strings.ReplaceAll` вместо `strings.Replace(s, old, new, -1)`

### По среде
- WSL/NTFS: `os.MkdirAll` на несуществующих корневых путях успешен — для тестов использовать трюк «файл как компонент каталога»
- Концы строк: всегда `sed -i 's/\r$//'` после создания файла
- `.gitignore`: всегда `/шаблон` для привязки к корню

## Значимое открытие: пробелы в проектировании Эпика 6

Во время ретроспективы обнаружено, что система управления знаниями (Эпик 6) имеет архитектурные пробелы, подтверждённые исследованием (20 источников, включая Anthropic, GitHub, Chroma Research).

**Исследование:** `docs/research/knowledge-extraction-in-claude-code-agents.md`

### 4 пробела, требующих дополнений:

1. **Нет валидации актуальности записей.** LEARNINGS.md не проверяет, что записанные правила ещё релевантны. У GitHub Copilot валидация по ссылкам на код даёт +7% к слиянию запросов (исследование Chroma/GitHub).

2. **Проблема оформления CLAUDE.md.** Содержимое оборачивается оговоркой «может быть, а может и не быть релевантным», что снижает следование инструкциям. Сжатие контекста уничтожает правила. Хуки жизненного цикла обходят обе проблемы.

3. **Монолитный формат.** Все знания в одном потоке. Исследование деградации контекста (Chroma, 18 моделей) показывает: атомарные разрозненные факты эффективнее связного текста.

4. **Нет метрик петли обратной связи.** «Больше ошибок → больше записей → больше контекста → больше ошибок» — риск задокументирован, но не измеряется.
