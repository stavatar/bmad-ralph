## Challenges

### Foundation Story Scale (7.1)

Story 7.1 (Metrics Foundation) modified 18 files and changed the Execute() signature — the most impactful single story in the project. Result: 7 findings (highest in epic), 2 new patterns discovered (metrics lifecycle completeness, metrics error-path recording).

### Agent Communication Reliability

Agents frequently went idle without receiving messages — required 2-3 resends per assignment. Reviewer and developer each needed one shutdown+respawn cycle. This is infrastructure friction, not process failure. Documented as standard operating procedure.

### High-AC Stories = Higher Risk

Story 7.8 (Similarity Detection, 9 AC) produced the only CRITICAL finding in the epic. Pattern: AC count correlates with finding density. Stories with >7 AC warrant extra review scrutiny or splitting.

## New Patterns Extracted (Epic 7)

5 patterns added to `.claude/rules/code-quality-patterns.md`:

1. **Metrics lifecycle completeness** — StartTask must always have matching FinishTask on all code paths (Story 7.1)
2. **Metrics recording on error paths** — failed sessions with parseable output must record metrics before retry (Story 7.1)
3. **Incremental metrics recording** — record after each measurement, not collect-then-record-once (Story 7.9)
4. **Double Finish() on metrics collector** — tests must use RunMetrics from Execute(), not call Finish() again (Story 7.9)
5. **Enum switch completeness** — all possible values must be handled, auto-generated values can't fall through (Story 7.10)
