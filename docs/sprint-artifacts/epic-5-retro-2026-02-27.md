## Challenges

### Findings Average Up: 5.0 → 6.2

Epic 4 target was < 4.0, actual was 6.2. Root cause: ALL 9 new patterns are NEW categories, not repeats of Epic 4 issues. Interactive I/O was uncharted territory:

1. Intermediate file state capture (Story 5.3) — tests checked call counts but AC required file content verification
2. Mock type co-location (Story 5.3) — related mocks scattered across files
3. Busy-wait polling sleep (Story 5.3) — tight spin without time.Sleep burns CPU
4. Action fall-through path testing (Story 5.4) — approve/skip share code path but only one tested
5. Mock API surface consistency (Story 5.4) — trackingGatePrompt and sequenceGatePrompt had different fields
6. Fixture location consistency (Story 5.4) — inline fixtures vs package-level const inconsistency
7. Post-loop processing guard (Story 5.5) — multiple break paths with different semantics need flag guard
8. Symmetric assertion depth (Story 5.5) — two tests covering same behavior at different depths
9. Table return-value field completeness (Story 5.6) — assert Action but not Feedback in table cases

**Conclusion:** findings average is not a fixable metric — each epic on new territory discovers new pattern categories. The important metric is: zero repeat patterns from previous epics.

### Edit Tool Friction

Story 5.5: Edit tool had trouble matching tab-indented Go code. Python workaround used. Tooling friction, not process issue.
