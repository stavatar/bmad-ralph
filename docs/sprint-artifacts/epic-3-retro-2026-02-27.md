**Что сделали:**

Разбили `.claude/rules/go-testing-patterns.md` (84 паттерна, 125 строк) на 6 тематических файлов + index:

| Файл | Правил | Glob (когда загружается) |
|------|--------|--------------------------|
| `test-naming-structure.md` | 10 | `*_test.go` |

| `test-mocks-infra.md` | 18 | `*_test.go` (включает CLI testing) |

| `go-testing-patterns.md` | — | index-only (17 строк, ссылки на sub-files) |

**Ключевое решение:** `code-quality-patterns.md` с glob `*.go` — паттерны вроде "stale doc comments" и "error wrapping consistency" должны загружаться при редактировании ЛЮБОГО Go файла, не только тестов. **Все файлы:** `.claude/rules/` (8 файлов: 6 topic + 1 index + wsl-ntfs.md)

### R2: PreToolUse checklist hook — DONE

**Проблема:** Правила о doc comments, error wrapping, assertions лежат в файлах. Агент может видеть их, может не видеть (compaction, context position). ngrok BMO [S35]: "ambient monitoring used only 2x in 60+ sessions."

**Что сделали:**

Создали `.claude/hooks/pre-edit-checklist.sh`:
- **Триггер:** PreToolUse на `Edit|Write`
- **Фильтр:** только для `*.go` файлов (не показывается для .md, .yaml, etc.)
- **Механизм:** читает `tool_input.file_path` из stdin JSON (через python3, т.к. jq не установлен в WSL), возвращает `hookSpecificOutput.additionalContext` с чеклистом
- **Чеклист (6 пунктов):**
  1. Doc comments still accurate after this change?
  2. Error wrapping consistent (same prefix for ALL returns)?
  3. No duplicate test cases (check existing table first)?
  4. go fmt needed after this edit?
  5. No scope beyond AC (YAGNI)?
  6. Inner error assertions present for ALL table cases?

**Важная деталь:** Hook использует `python3 -c` вместо `jq` для парсинга JSON, потому что jq не установлен в WSL environment. Python3 (3.10.12) доступен.

Добавили в `.claude/settings.json`:
```json
"PreToolUse": [{"matcher": "Edit|Write", "hooks": [{"type": "command", "command": "\"$CLAUDE_PROJECT_DIR\"/.claude/hooks/pre-edit-checklist.sh"}]}]
```

### R3: PostToolUse CRLF auto-fix — DONE

**Проблема:** Правило "после Write делай sed" было в CLAUDE.md, critical-rules.md, wsl-ntfs.md, SessionStart hook — и всё равно нарушалось в 2/11 stories. Это deterministic задача, которая не должна зависеть от "решения" агента.

**Что сделали:**

Создали `.claude/hooks/fix-crlf.sh`:
- **Триггер:** PostToolUse на `Edit|Write`
- **Действие:** читает `tool_input.file_path` из stdin JSON (python3), проверяет что файл — text (через `file` command), выполняет `sed -i 's/\r$//'`
- **100% deterministic:** агент вообще не участвует в решении

Добавили в `.claude/settings.json`:
```json
"PostToolUse": [{"matcher": "Edit|Write", "hooks": [{"type": "command", "command": "\"$CLAUDE_PROJECT_DIR\"/.claude/hooks/fix-crlf.sh"}]}]
```

**Убрали пассивное правило из:**
- `CLAUDE.md` — секция "Critical: Line Endings" обновлена: "CRLF auto-fixed by PostToolUse hook"
- `.claude/critical-rules.md` — правило #12 (ручной sed) удалено, заменено на правило про code-review protocol
- `.claude/rules/wsl-ntfs.md` — обновлено: "auto-fixed by PostToolUse hook"
